## THE FUNNEL RULES

### RULE 1: C2 and C3 REPORT TO C1
- C2 does work → Reports to C1
- C3 does work → Reports to C1
- C1 summarizes all three

### RULE 2: Each Computer Makes ONE Output
- All 7 instances on CP1 → ONE CP1_OUTPUT.md
- All 7 instances on CP2 → ONE CP2_OUTPUT.md
- All 7 instances on CP3 → ONE CP3_OUTPUT.md

### RULE 3: All Computers Report to ONE Place
- CP1_OUTPUT.md → Goes to sync folder
- CP2_OUTPUT.md → Goes to sync folder
- CP3_OUTPUT.md → Goes to sync folder

### RULE 4: Commander Reads ONE Place
- `G:\My Drive\TRINITY_COMMS\sync\`
- That's it. Everything funnels there.

---

## REPORTING FORMAT

Every instance reports to their C1:

```
INSTANCE: [C1/C2/C3] [Cloud/Terminal/Desktop]
COMPUTER: [CP1/CP2/CP3]
I DID: [one sentence]
I MADE: [file names]
I NEED: [blockers]
```

C1 of each computer summarizes into:

```
COMPUTER: [CP1/CP2/CP3]
SUMMARY: [what the whole computer accomplished]
FILES: [what was created]
BLOCKERS: [what's stopping progress]
NEXT: [what happens next]
```
